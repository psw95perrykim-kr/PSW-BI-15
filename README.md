# 인스턴트 디제이 (Instant DJ)

모바일에서 유튜브 음악을 간단히 디제잉하는 웹앱. 조수석에서 다음 곡을 자연스럽게 넘기려고 만들었다.

## 주소

| | 주소 | 용도 |
|---|---|---|
| **본편** | **https://psw95perrykim-kr.github.io/PSW-BI-15/** | 턴테이블 DJ 콘솔. **평소엔 이걸 연다.** |
| 검증 | https://psw95perrykim-kr.github.io/PSW-BI-15/spike.html | 기기별 재생 능력 점검용. 턴테이블 UI가 아니라 테스트 버튼과 로그만 있는 페이지. |

> `spike.html`은 디자인이 밋밋해서 "예전 UI"로 오해하기 쉽다. 본편이 아니라 검사 도구다.

## 쓰는 법

1. 크레이트에 유튜브 링크를 붙여넣고 **추가**
2. 목록에서 **A** / **B** 를 눌러 각 덱에 올린다
3. 플래터를 **탭** 하면 재생/정지, **돌리면** 스크래치·탐색 (한 바퀴 = 6초)
4. **⇄ 넘기기** 한 번이면 죽어있는 쪽 덱을 자동 재생하고 2/4/8초에 걸쳐 크로스페이드

LCD 잔여 시간이 30초 이하로 떨어지면 빨갛게 변한다 — 다음 곡 올리라는 신호.

## 왜 GitHub Pages인가

유튜브 iframe API는 `http://<IP>:<port>` 같은 오리진의 임베드를 거부한다.
`getVideoData()`가 `isPlayable: false, errorCode: "auth"` 를 돌려주는데
**`onError` 이벤트는 뜨지 않아서** 그냥 "눌러도 반응 없음"으로만 보인다.
HTTPS 도메인이 필요해서 Pages로 띄운다. (`localhost` 는 예외라 로컬 개발은 그대로 된다.)

## 파일

- `index.html` — 본편 DJ 콘솔
- `spike.html` — iOS 기술 검증 페이지 (동시재생·볼륨제어·크로스페이드·재생속도·시크 지연)

`.git` 을 iCloud 밖에 두려고 분리했다.
